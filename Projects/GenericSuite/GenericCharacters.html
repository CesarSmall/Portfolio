Generic Characters

Quick and simple characters for Unreal Engine

What is it?

Generic Characters is a plugin I've developed for Unreal Engine that provides an expansion on the base character templates providing expected extra behaviours such as crouching and sprinting in a fully replicated Character class with easily extendable input mappings via a custom input mapping system that I've created.

The idea is that the user can install the plugin and have a character that works with them for setting up inputs and basic movement for quick prototyping of their game. Mostly useful for people who are maybe not super familiar with Enhanced Input or just want to get something working quickly.

It currently includes:

  • Customizable inputs
  • Support for first and third characters
  • Multiplayer input support
  • Multiple mapping context swaps through InputMapping registration and extendable Input Mode enums

    • How to use it

    Once the plugin is installed, using it is as simple as creating a new Blueprint class based on the Generic Character class provided by the plugin and setting up the desired input mappings in the Input Mapping Data asset also provided.

    From there the rest of the setup is up to the user in their project, which is typically by setting the pawn class in their game mode.

    Examples

    In this example I show part of the system I've created to simplify adding and removing input actions and mappings to the player. Please note that this plugin is still in a very early stage.

    Action binding logic for the Character System using Enhanced Input:

    void AGPlayerController::RegisterActionBinding(UEnhancedInputComponent* EnhancedInputComponent, const FGenericInputDefinition& Action)
{
	AGCharacter* CharacterInstance = Cast(GetPawn());
	if (!CharacterInstance)
	{
		UE_LOG(LogTemp, Warning, TEXT("AGPlayerController::RegisterActionBinding - No Character on %s"), *GetName());
		return;
	}
	
	UInputAction* InputAction = Action.Action;
	FEnhancedInputActionEventBinding& Binding = EnhancedInputComponent->BindAction(
		InputAction, Action.TiggerEvent, CharacterInstance, Action.Function);

	ActionBindings.Add(InputAction, &Binding);
}
