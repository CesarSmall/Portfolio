Generic Crafting

Simple crafting system with no hassle in Unreal Engine

What is it?

It is a simple crafting and processing system that expands GenericStorage and allows you to define recipes and add these items to a station of any kind and construct items from others either with or without a crafting time.

How to use it

To use the system, simply create a recipe DataAsset. With that Data Asset you can define the input and output items of the recipe as well as the crafting time and the station where it can be crafted.
The crafting system itself is contained in an actorcomponent and contains a storage component from GenericStorage inside as well. With this you can set it all up according to the properties. The component has overridable functions for setting custom behaviour for crafting start, finish, and cancel.

Examples

In the example below I show off the logic behind the processor for the crafting component. It showcases the starting of the processor component.

Start processing code snippet:

void UGProcessorComponent::TryStartProcessing(UGProcessorRecipe* Recipe)
{
	//We shouldn't do this on the client, call server to do check
	if (!GetOwner()->HasAuthority())
	{
		Server_TryStartProcessing(Recipe);
		return;
	}

	//Make sure we have a valid recipe index
	if (!Recipes.Contains(Recipe))
	{
		if (bDebugText)
		{
			UE_LOG(LogGProcessing, Error, TEXT("INVALID RECIPE!"))
		}
		OnInvalidRecipeIndex.Broadcast(GetOwner(), CurrentRecipeIndex);
		return;
	}

	//Since we know it's valid, let's fetch and store the recipe from its index
	UGProcessorRecipe* CurrentRecipe = Recipes[CurrentRecipeIndex].Get();

	//Make sure we're not already processing
	if(bIsProcessing)
	{
		if (bDebugText)
		{
			UE_LOG(LogTemp, Warning, TEXT("Processing already started!"))
		}
		//If we've reached this at any point it means we failed to start the process properly
		OnFailedToStartProcessing.Broadcast(GetOwner(), CurrentRecipe);
		return;
	}

	//Try and gather the necessary items from input storage
	if (!PullItemsFromInputStorage(CurrentRecipe, HeldInputItems))
	{
		PutItemsIntoInputStorage(HeldInputItems); //Put the items we took out back into the storage container
		UE_LOG(LogGProcessing, Error, TEXT("We do not have all of the items necessary to start the processing!"))
		OnFailedToStartProcessing.Broadcast(GetOwner(), CurrentRecipe);
		return;
	}

	//We shouldn't start the processing if we don't have the appropriate space to store the output stuff.
	//That would just make the output disappear
	if (!HasEnoughSpaceForOutput())
	{
		PutItemsIntoInputStorage(HeldInputItems); //Put the items we took out back into the storage container
		UE_LOG(LogGProcessing, Error, TEXT("Not enough space in output to start processing!"))
		OnFailedToStartProcessing.Broadcast(GetOwner(), CurrentRecipe);
		return;
	}

	//We've passed all of the checks and can now start processing
	if (bDebugText)
	{
		UE_LOG(LogGProcessing, Display, TEXT("STARTING PROCESSING"))
	}

	//Set the processing time and enable tick so that we can actually process
	RemainingTime = GetCurrentRecipe()->TimeToProcess;
	SetComponentTickEnabled(true);
	
	OnStartItemProcess.Broadcast(GetOwner(), CurrentRecipe);//Send out a broadcast to let other systems know that we've successfully started our processing
}
